# Implementation Plan: Images Panel in Left Sidebar

## Goal
Add an "Images" tab/button to the left sidebar (`ConversationSidebar`) that shows all images generated by Sophia, alongside the existing conversations list. Users can browse, click to view full-size, and see the prompt that generated each image.

---

## Current State Analysis

### What exists today:
1. **Backend**: `POST /api/images/generate` — generates image, stores as `UserFile`, returns `{id, filename, mime, url}`
2. **Backend**: `GET /api/files/{id}/raw` — serves raw bytes (works for images)
3. **Backend**: `sophia/image_gen/generator.py` — HF FLUX.1-schnell, returns JPEG bytes
4. **Frontend**: `Composer.tsx` has an `ImageIcon` button that opens an inline prompt to generate images
5. **Frontend**: `ChatWorkspace.tsx` has `addGeneratedImage()` that appends `![generated](file:{id})` as a Sophia message
6. **Frontend**: `ImageGenerateOut` type exists in `web/lib/types.ts`
7. **DB**: `UserFile` table stores generated images with `mime_type`, `stored_path`, `user_id`, `created_at`
8. **Frontend**: `ConversationSidebar.tsx` — single-purpose: search + list conversations, grouped by date

### What's missing:
- No endpoint to **list** generated images for a user
- No UI in the sidebar to **browse** images outside of chat context
- No **full-size image viewer** (lightbox)
- Images are only visible inside chat messages after generation, not as a persistent gallery

---

## Architecture Decision: Tabbed Sidebar

The sidebar becomes a **two-tab panel**:
- **Conversations tab** (existing, default) — search, new, list, rename, delete
- **Images tab** (new) — thumbnail grid of generated images, click to view full-size

A toggle at the top of the sidebar switches between tabs. The conversations button header becomes a tab bar.

## Files to Modify

### 1. New Backend Endpoint
**File**: `sophia/app/routers/images.py`
```
GET /api/images  →  list all generated images for the current user
                   returns ImageGenerateOut[] ordered newest first
```
- Owner-scoped: `session.query(UserFile).filter(UserFile.user_id == user.id, UserFile.mime_type == "image/jpeg")`
- Order by `created_at.desc()`

### 2. New DB Service Function
**File**: `sophia/db/service.py`
```python
def get_user_images(session: Session, user_id: int) -> list[UserFile]:
    """Return all image UserFiles for a user, newest first."""
    return (
        session.query(UserFile)
        .filter(UserFile.user_id == user.id, UserFile.mime_type.startswith("image/"))
        .order_by(UserFile.created_at.desc())
        .all()
    )
```

### 3. New BFF Route
**File**: `web/app/api/images/route.ts` (new)

  const res = await forward("/api/images", { method: "GET" });

## Data Flow

```
User clicks "Images" header button
  → ChatWorkspace sets sidebarTab = "images", sidebarOpen = true
  → ConversationSidebar renders ImageGallery
  → ImageGallery mounts → fetches /api/images (BFF → FastAPI)
  → FastAPI: get_user_images(session, user.id) → ordered list
  → Renders thumbnail grid
  → User clicks thumbnail → ImageLightbox opens with full-size image

User generates image via Composer
  → addGeneratedImage(image) called
  → Chat message updated
  → ImageGallery refetches /api/images (SWR or explicit refresh)
  → New image appears at top of grid
```
